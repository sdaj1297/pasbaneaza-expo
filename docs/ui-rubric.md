# UI Rubric

Use this rubric before accepting any UI change.

Score each category from 0 to 2.

- 0: fails the requirement
- 1: partially meets it
- 2: clearly meets it

Minimum acceptable score: 14 out of 18.

## 1. Product Hierarchy

The schedule and event utility are visible before decorative content.

## 2. Schedule Scanability

Users can quickly read time, host, title, and location.

## 3. Anjuman vs Community Distinction

Committed Anjuman programs are visually distinct from approved community programs.

## 4. Flyer Preservation

Special event flyers are supported without hijacking the homepage.

## 5. Visual System

Spacing, type, color, radius, shadows, and borders use a deliberate system.

## 6. Brand Taste

The UI feels specific to Pasban-e-Aza and respectful to the subject matter.

## 7. Accessibility

Contrast, focus, hit targets, and text sizing are usable.

## 8. Mobile First

The mobile experience is not an afterthought.

## 9. Native-App Portability

The design language can translate to Expo/iOS/Android later.

## Hard Fails

Reject the UI regardless of score if any of these are true:

- mojibake or broken text is visible
- the logo is missing from primary navigation
- the schedule appears below decorative quote/hero content
- prayer times dominate the homepage top
- Anjuman committed programs cannot be identified quickly
- text overlaps or clips on mobile
- the design looks like a generic template with Pasban colors pasted on
